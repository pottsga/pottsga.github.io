Summary

Styling your website is one of the most important steps in the web design processes. This presentation layer allows for the users you are seeking to reach the ability to interact with the various pieces of content. While writing pure CSS gives the web developer the most control of the exact styles, for the general layout of the web application and utilization of common components that are often used in the web, CSS Frameworks become a very useful tool to have in your tool belt.

Bootstrap has recently been upgraded to v5.0. While technically in a beta version, this release is ready for use in your modern web applications today. Notable features of this release are the removal of the necessity of jQuery, the introduction of Bootstrap's own icon library, and optimizations on the already-robust component and layout library that the CSS Framework boasts.

Introduction

Layout

Layout within Bootstrap is generall accomplished with <div class="row"></div> and <div class="col"></div> elements of some sort.

An example 3-column layout can be accomplished by the following code. Bootstrap allows for flexible columns that will take up the amount of space they are given but also allow the developer to specify explicitly how many units the column should span. The following code is equivalent:

  <style>
    .border {
      border: 1px solid grey;
    }
  </style>
  <div class="row">
    <div class="col border">1</div>
    <div class="col border">2</div>
    <div class="col border">3</div>
  </div>

  <style>
    .border {
      border: 1px solid grey;
    }
  </style>
  <div class="row">
    <div class="col-4 border">1</div>
    <div class="col-4 border">2</div>
    <div class="col-4 border">3</div>
  </div>

Adding a sm, md, lg, or xl descriptor to a col helps that column to responsively resize itself depending on the size of the viewport that the column is within. These breakpoints, as they are called, are limited by the following bounds:

Forms

Basics

Forms are also a critical component to style correctly. Users that navigate to a website expect that the forms within that website be consistent and professional looking. Bootstrap allows for this task to be a breeze and responsive forms, even with validation, are trivial.

By adding the .form-control class to <input> elements, those elements are styled with better defaults than what is delivered in most browsers. Adding a <div class="form-group"></div> element as the parent of the <input> and <label> elements allows those elements to span the entire width of the container they are in. By default, by simply adding this class to an input with a label, this is what the form looks like:

<form action="#">
  <div class="form-group">
    <label for="">First Name</label>
    <input type="text" class="form-control" name="first_name" id="first_name">
  </div>
</form>

Automatic Required Asterisk

While this is not specific to Bootstrap, a notable mention about forms is that it is ideal for required <input> elements to have styling associated with them that shows that they are required visually. This is usually done in the form of an asterisk next to the end of the label's text. This can, and should, be done automatically so the developer does not have to think about it while developing the form. To do this, some CSS and JS is needed to be injected, which should be applied globally on some top-level global files.

.required::after {
  color: salmon;
  content: " *";
}

/**
 * Add a required class to labels whose inputs, selects, or textareas
 * have a required attribute on them
 */
function addRequiredClassToLabels() {
  var requiredInputSelectTextareas = document.querySelectorAll(
    'input[required]:not([type="checkbox"]),select[required],textarea[required]'
  );
  for (var i = 0; i < requiredInputSelectTextareas.length; i++) {
    var requiredInputSelectTextarea = requiredInputSelectTextareas[i];
    var parent = requiredInputSelectTextarea.parentElement;
    var label = parent.querySelector('label');
    if (label !== null) {
      label.classList.add('required');
    }
  }
}

This simple code allows any <input> with a sibling <label> element within a parent container to have a red asterisk added to it.
